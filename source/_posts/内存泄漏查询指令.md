valgrind --leak-check=full ./app_name

## 一、真正判断内存泄露只看这 3 行
你这份报告里最关键的是：

```plain
LEAK SUMMARY:
   definitely lost: 0 bytes in 0 blocks   ✅
   indirectly lost: 0 bytes in 0 blocks   ✅
     possibly lost: 592 bytes in 1 blocks
   still reachable: 4,731,324 bytes
```

### ✅ 结论含义
| 类型 | 是否内存泄露 | 解释 |
| --- | --- | --- |
| definitely lost | ❌ 否 | **真正的内存泄露（你这里是 0）** |
| indirectly lost | ❌ 否 | |
| possibly lost | ⚠️ 几乎可忽略 | 第三方库常见 |
| still reachable | ❌ 否 | 程序退出时仍可访问 |


👉 **Valgrind 官方定义：只有 **`**definitely lost**`** 才是必须修的泄露**
